# 3D Viewer v2.0

Кроссплатформенный учебный проект на C++20 для просмотра и манипуляции трёхмерными моделями.  
Реализация базируется на архитектуре **MVC** с использованием паттернов **Facade** и **Strategy** для управления трансформациями.

Доступны цели в CmakeLists - all, install, uninstall, clean, dvi, dist, test, gcov_report

> **О проекте:** Универсальный просмотрщик 3D-моделей с интерактивным управлением камерой, трансформацией объектов и полной поддержкой OpenGL для высокопроизводительного рендеринга.

Модели для тестов лежат в папке src/obj

## О проекте

3D Viewer v2.0 состоит из:

- **Ядра парсера моделей** — загрузка и обработка 3D-моделей из файлов;
- **Системы трансформаций** — вращение, перемещение и масштабирование объектов;
- **Графического интерфейса на Qt6** — интерактивное управление сценой;
- **OpenGL-рендерера** — эффективная визуализация 3D-геометрии;
- **Unit-тестов** — полное покрытие логики моделирования с помощью GoogleTest.

## Основные возможности

### Загрузка и парсинг моделей

- Поддержка стандартного формата OBJ;
- Автоматическая нормализация модели при загрузке;
- Центрирование объекта в пространстве;
- Автоматический расчёт экстремумов (границ объекта);
- Информация о количестве вершин и граней.

### Трансформации объектов

- **Вращение** — по всем трём осям (X, Y, Z) с плавной интерполяцией;
- **Перемещение** — смещение в 3D-пространстве;
- **Масштабирование** — контролируемое увеличение/уменьшение в процентах;
- **Реализация паттерна Strategy** — гибкая система применения различных трансформаций.

### Управление камерой

- **Углы обзора** — контролируемое вращение камеры;
- **Расстояние** — динамическое приближение/отдаление;
- **Интуитивное управление** — мышь для положения, клавиатура для масштабирования.

### Логирование и отладка

- **Система логирования** — запись событий приложения в файл и консоль;
- **Уровни логирования** — DEBUG, INFO, WARNING, ERROR;
- **Трассировка ошибок** — полный контроль над ошибками парсинга и рендеринга.

## Архитектура

Проект следует паттерну **Model-View-Controller (MVC)** с дополнительными паттернами:

- **Model** (`models/model.h`) — парсинг файлов, хранение геометрии, трансформации;
- **View** (`views/glWidget.h`, `views/mainWindow.h`) — рендеринг OpenGL, интерфейс Qt;
- **Controller** (`controllers/controller.h`) — синхронизация между Model и View;
- **Facade** (`facade/viewerFacade.h`) — предоставляет упрощённый интерфейс к сложной системе;
- **Strategy** (`models/strategy.h`) — различные типы трансформаций (ротация, сдвиг, масштабирование).

### Принципы разработки

- Полное разделение бизнес-логики от кода интерфейса;
- Использование умных указателей (`std::unique_ptr`, `std::shared_ptr`);
- Применение сигналов и слотов Qt для коммуникации компонентов;
- Паттерн **Strategy** для extensible системы трансформаций;
- Паттерн **Facade** для упрощения работы с моделями;
- Полное логирование всех операций;
- Unit-тесты для критических компонентов.

## Поддерживаемые форматы файлов

### OBJ (Wavefront)

- ✅ Вершины (`v x y z`)
- ✅ Грани (`f v1 v2 v3`)
- ✅ Комментарии (`#`)
- ✅ Нормали (`vn`) — парсятся, но не используются в текущей версии
- ✅ Текстуры (`vt`) — парсятся, но не используются в текущей версии

Пример загрузки:

```cpp
Model model;
model.parseFile("path/to/model.obj");

std::cout << "Вершин: " << model.vertices().size() << std::endl;
std::cout << "Граней: " << model.faces().size() << std::endl;
```

## Система логирования

Все операции приложения логируются в файл `3dviewer.log`:

```cpp
s21::Logger& logger = s21::Logger::getInstance();
logger.setLogFile("3dviewer.log");
logger.setLogLevel(s21::LogLevel::DEBUG);
logger.info("Приложение запущено");
logger.error("Ошибка при загрузке модели");
```

**Уровни логирования:**

- `DEBUG` — подробная отладочная информация
- `INFO` — информационные сообщения
- `WARNING` — предупреждения
- `ERROR` — критические ошибки

## Кодовые стандарты

Проект разработан с соблюдением:

- **Google C++ Style Guide** — именование, отступы, документирование;
- **C++20 best practices** — современные конструкции языка;
- **Qt best practices** — правильное использование сигналов/слотов, памяти;
- **OpenGL best practices** — эффективная визуализация, минимизация draw calls.

## Требования к коду

- Все функции должны быть покрыты unit-тестами;
- Минимальное покрытие кода: **≥ 80%**;
- Все заголовочные файлы должны иметь include guards;
- Используется C++20 (constexpr, concepts, ranges где применимо).

## Известные ограничения

- Максимальный размер модели: ограничен памятью видеокарты;
- OpenGL версия: требуется 3.3+ для полной функциональности;
- Поддержка только OBJ-формата (расширение в планах);
- Текстуры и нормали не поддерживаются в текущей версии.
